## Files Created

### `backend/Dockerfile`
- Multi-stage build: `python:3.11-slim` builder installs deps, runtime stage copies only what's needed
- Non-root `appuser` for security
- Exposes port 8000

### `frontend/Dockerfile`
- Stage 1: `node:20-alpine` runs `npm ci && npm run build`
- Stage 2: `nginx:alpine` serves `dist/` with SPA fallback (`try_files $uri /index.html`)
- `VITE_API_BASE_URL` injected as build ARG (set to `/api/v1` — relative path works because nginx proxies)

### `nginx/nginx.conf`
- Single HTTP `server` block active (no domain = no SSL yet)
- `/api/` → proxied to `backend:8000` (120s timeout for long Celery polls)
- `/webhook/telegram` → proxied to `backend:8000`
- `/health` → proxied to `backend:8000/api/v1/health/`
- `/*` → proxied to `frontend:80`
- Full HTTPS `server` block pre-written and commented out — activate with Certbot when domain arrives

### `docker-compose.prod.yml`
Key differences from dev compose:

| Service | Dev | Prod |
|---|---|---|
| `backend` command | `uvicorn ... --reload` | `gunicorn ... -k UvicornWorker --workers 2` |
| `celery-worker` concurrency | 4 | 2 |
| `openclaw-worker` | included | **removed** (external VPS) |
| `frontend` | not present | nginx static build |
| `nginx` | not present | public-facing reverse proxy |
| `migrate` | not present | init container runs `alembic upgrade head` |
| Postgres port | `5432:5432` exposed | internal only |
| Memory limits | none | db=none, backend=600M, celery=400M, beat=150M, frontend=100M, nginx=50M |
| Healthchecks | none | db + redis both healthchecked; migrate + backend wait on them |

### `deploy.sh`
One-command deploy for Hostinger VPS:
1. Checks `backend/.env` exists (exits with instructions if not)
2. `git pull origin main`
3. `docker compose -f docker-compose.prod.yml down --remove-orphans`
4. `docker compose -f docker-compose.prod.yml build --no-cache`
5. `docker compose -f docker-compose.prod.yml up -d`
6. Polls `http://localhost/health` up to 15 times before declaring success/failure

## Architecture on VPS

```
Internet (port 80)
      |
   [nginx:alpine] ─── /api/*             → backend:8000 (gunicorn, 2 workers)
                  ─── /webhook/telegram  → backend:8000
                  ─── /*                 → frontend:80 (nginx static)

   [backend] ←→ [redis:7-alpine] ←→ [celery-worker, concurrency=2]
      |
   [db: postgres:15-alpine] (internal only, no host port)
   [celery-beat]
   [migrate] (init container, exits after alembic upgrade head)
```

Total estimated RAM usage at idle: ~1.4GB (leaves ~600MB headroom on 2GB VPS)

## Deployment Commands (first time on VPS)

```bash
# Install Docker
sudo apt update && sudo apt install -y docker.io docker-compose-plugin git
sudo usermod -aG docker $USER && newgrp docker

# Clone and configure
git clone <repo-url> salesinject && cd salesinject
cp .env.example backend/.env
nano backend/.env   # fill: POSTGRES_PASSWORD, SECRET_KEY, BOT_TOKEN, OPENCLAW_URL

# Deploy
chmod +x deploy.sh && ./deploy.sh
```

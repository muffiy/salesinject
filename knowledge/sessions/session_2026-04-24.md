## What Was Built This Session

### 1. `backend/app/tools/paperclip_tools.py`
Five `@tool`-decorated functions forming the scout pipeline:

| Tool | Service Called | Output |
|---|---|---|
| `scout_influencers(niche, location)` | `exa_service.find_influencers()` | `List[Dict]` |
| `analyze_and_rank(influencers_data, niche)` | `agent_zero_service.analyze_influencers()` | `str` report |
| `save_scout_results(user_id, task_id, influencers, report, ad_copy, db)` | `paperclip_service.write_paperclip_items()` | `{rows_written}` |
| `save_scout_report(user_id, influencers, niche, db)` | Writes to `scout_reports` table | `{record_id}` |
| `notify_telegram(chat_id, message)` | `telegram_service.send_message()` via `asyncio.run()` | `{status}` |

A lightweight custom `@tool` decorator was used (no LangChain dependency). It registers each function in `_TOOL_REGISTRY` with `.is_tool`, `.tool_name`, `.tool_description` attributes — drop-in compatible with a future LangChain swap.

## Key Design Decisions

1. **`@tool` decorator is custom, not LangChain** — avoids heavy dependency, swappable in 2 lines.
2. **All tools use lazy imports** (inside function body) — prevents circular import at module load.
3. **Single `db.commit()` in orchestrator** — all tool flushes are intermediate; the agent owns the transaction.
4. **`notify_telegram` is `asyncio.run()` wrapped** — `telegram_service.send_message` is `async`, Celery workers are sync. Safe bridge with silent failure.
5. **`save_scout_report` generates map coordinates** — adds lat/lng fallback scatter near Tunis `[36.8065, 10.1815]` when Exa doesn't return geo data.
